● 多台电脑使用一个Github账户

一、work电脑

我是先在公司电脑提交项目到Github上的，具体过程点这里

二、home电脑

重点，回家后……

1、首先将home电脑和Github绑定

用户名绑定

用户邮箱绑定

2、生成新的SSH密匙

将新的SSH设置到Github账户上，注意：是新增SSH，不是替换，work电脑的SSH还是存在的

3、然后将Github上想要修改的或者继续开发的项目clone到home电脑上

完美，home电脑上已经存在Github上的项目了，然后想要修改或者继续开发都行

4、完成修改或者开发后，需要在home电脑上提交修改或者开发后的项目文件

提交成功

三、再次到work电脑

远程的代码已经发生改变，所以首先要将work代码和Github的项目代码同步

1、将远程最新的代码下载到分支上

2、将分支的代码和本地代码合并

完成。
这样home电脑修改或者开发的代码同样到了work电脑。
继续开发，提交，同步，开发，提交，同步……

多台电脑也一样

同步代码时，如果想要查看本地代码和远程代码的差别

知识普及：ssh是什么?
“ssh是一种安全的传输模式，github要求推送代码的用户是合法的，所以每次推送时候都要输入账号密码，用以验证你是否为合法用户，为了省去每次都要输入密码的步骤，采用shh公钥，密钥；也就是你说的sshkey来验证你是否为合法用户，在你的电脑生成了一个唯一的ssh公钥和私钥，公钥放到github上面，当你推送的时候，git就会匹配你的私钥是否跟github上面的公钥是配对的，正确就认为你是合法的，允许推送。sshkey可以理解为是你的身份标识，放在github上面表明你是这个项目的一个开发人员，但是别人是可以截获的，你本机的私钥别人就无法截获，sshkey就可以保证每次传输都是安全的。”
